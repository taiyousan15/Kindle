# 要件定義書 — Kindleリサーチ分析システム

**バージョン**: 1.0.0
**作成日**: 2026-03-06
**ステータス**: 承認済み（リサーチフェーズ完了）

---

## 1. プロジェクト概要

### 1.1 背景・目的

日本の電子書籍市場（2024年: 6,703億円 / 前年比+3.9%）において、amazon.co.jpのKindle本専用リサーチツールは存在しない（ブルーオーシャン）。

**目的**: Kindle本出版者が売れる本を科学的に設計できるよう、BSR・キーワード・ジャンルトレンド・表紙傾向を自動収集・分析し、売上予測とコンテンツ提案を提供するシステムを構築する。

### 1.2 スコープ

| 対象 | 内容 |
|------|------|
| マーケット | amazon.co.jp（日本市場専用） |
| 書籍種別 | Kindle電子書籍（ASIN: B00xxx形式） |
| ユーザー | KDP出版者・個人著者・出版社 |
| 除外 | 紙書籍・Amazon.com（英語市場）・KENP直接取得 |

## 2. ステークホルダー要件

### 2.1 ペルソナA — KDP個人出版者（メインユーザー）

**特徴**: 月1〜3冊出版。マーケティング知識は中程度。Excelは使えるがSQLは書けない。

**主要ニーズ**:
- 今何が売れているかを「今日・今週・今月・半年」の時間軸で把握したい
- 売れているタイトルのパターンを見て、自分の本に応用したい
- 自分のジャンルで競合が少なく需要のあるキーワードを見つけたい
- 表紙の傾向を視覚的に確認したい

**受け入れ条件**:
- ダッシュボードに5分以内でジャンル分析結果が表示される
- キーワード難易度・推定検索ボリュームが数値で確認できる
- タイトル・説明文の改善案がAIから提示される

### 2.2 ペルソナB — 出版社・プロデューサー（パワーユーザー）

**特徴**: 複数著者を管理。月10〜30冊の出版計画。データドリブンな意思決定を好む。

**主要ニーズ**:
- ジャンル別の競合強度と収益ポテンシャルを定量的に比較したい
- 半年〜1年間のBSRトレンドから季節性・需要サイクルを把握したい
- 著者ごとに担当ジャンルの市場状況をレポートで共有したい
- 新しいジャンルへの参入タイミングを予測したい

**受け入れ条件**:
- CSVエクスポートでExcel分析可能
- ジャンル別レポートをPDF/Markdownで出力できる
- 複数ジャンルを横断比較できる

### 2.3 ペルソナC — システム管理者（運用者）

**特徴**: データ収集・APIコスト管理・障害対応を担当。

**主要ニーズ**:
- APIコストを月次で把握・制御したい
- データ収集スケジュールの管理・監視
- Creators API廃止（2026/4/30）への事前移行

**受け入れ条件**:
- 月次コスト上限アラートが設定できる
- 収集ジョブの成功/失敗がログで確認できる
- PA-API→Creators API切り替えが設定で対応可能

## 4. 非機能要件

### 4.1 パフォーマンス

| 要件 | 目標値 |
|------|--------|
| ダッシュボード初期表示 | < 3秒 |
| キーワード分析レスポンス | < 5秒（キャッシュあり） |
| バッチ収集（1,000 ASIN） | < 30分 |
| BSR予測計算 | < 10秒 |

### 4.2 可用性・信頼性

- システム稼働率: 99.5%以上（月次）
- データ収集失敗時: リトライ3回 → 失敗記録 → 次回バッチで補完
- Keepa API障害時: 最終取得データから補間表示（警告表示あり）

### 4.3 セキュリティ

- APIキーは環境変数管理（`.env`、Gitに含めない）
- ユーザー認証: JWT + Refresh Token（Phase2以降でのSaaS化時）
- HTTPS必須
- Amazon ToS遵守: スクレイピング禁止・Creators API利用規約準拠

### 4.4 コスト制約

| フェーズ | 月額上限 | 必須コスト |
|---------|---------|-----------|
| Phase 1 | $150 | Keepa $54 + MerchantWords $29 + VPS $10 + AI $8 = $101 |
| Phase 2 | $300 | + Helium10 $129 + Vercel $20 |
| Phase 3 | $500 | + TimescaleDB Cloud $100 + K8s $150 |

### 4.5 法的要件

- Amazon Creators API利用規約の完全遵守
- Keepa API Terms of Service準拠
- MerchantWords利用規約準拠
- 書影画像: Creators API提供URLのみ参照（直接ダウンロード・ホスト禁止）
- PA-API廃止対応期限: **2026年4月30日**（必須）

## 5. ユーザーストーリー（Epic別）

### Epic 1: キーワードリサーチ

```
US-001: KDP出版者として、「AI活用」というキーワードの
        月間推定検索ボリューム・競合度・関連キーワードを
        5分以内に確認できるようにしたい。
        理由: どのキーワードで本を出すべきか判断したいから。

US-002: KDP出版者として、自分のジャンルで競合が少なく
        需要のあるニッチキーワードを自動発見してほしい。
        理由: レッドオーシャンを避けた出版戦略を立てたいから。
```

### Epic 2: ジャンルトレンド把握

```
US-003: KDP出版者として、「自己啓発」ジャンルで
        過去30日間に急上昇した本の共通キーワードを把握したい。
        理由: 旬のトレンドに乗ったコンテンツを作りたいから。

US-004: 出版社プロデューサーとして、5ジャンルの
        TrendScoreを横断比較して次の企画を決めたい。
        理由: リソースをROIの高いジャンルに集中させたいから。
```

### Epic 3: タイトル最適化

```
US-005: KDP出版者として、考えたタイトル案を入力して
        AIから改善案と期待CTRスコアをもらいたい。
        理由: 表示された瞬間にクリックされるタイトルを作りたいから。

US-006: KDP出版者として、売れているタイトルのパターン
        （構造・文字数・感情語使用）を視覚的に確認したい。
        理由: 成功パターンを自分の本に応用したいから。
```

### Epic 4: 表紙分析

```
US-007: KDP出版者として、「ビジネス書」ジャンルの
        表紙傾向（配色・フォントスタイル・構図）を
        ギャラリー形式で確認したい。
        理由: 売れている表紙のデザインパターンを参考にしたいから。
```

### Epic 5: 売上予測

```
US-008: KDP出版者として、BSR 5,000の本が
        月に何冊売れているかを推定値+誤差範囲付きで知りたい。
        理由: 収益シミュレーションをしたいから。
```

## 6. 受け入れ条件（Definition of Done）

### Phase 1 MVP

- [ ] キーワード検索 → 推定ボリューム・競合度表示（10秒以内）
- [ ] ジャンル別ベストセラー100件のBSR一覧表示
- [ ] 時間軸フィルタ（今日・1週間・30日・半年）
- [ ] タイトル入力 → AIスコア+改善案表示
- [ ] 表紙分析（ジャンル選択 → 傾向レポート生成）
- [ ] BSR→推定販売数の変換計算
- [ ] 検索ボリュームは必ず「推定値」「信頼度★」と明示
- [ ] PA-API廃止前にCreators APIへ移行完了（〜2026/4/30）

### Phase 2 成長期

- [ ] SaaS化（ユーザー登録・課金・ダッシュボード）
- [ ] Helium10クロスチェック統合
- [ ] LightGBMによるBSR予測モデル（±15%精度）
- [ ] CSVエクスポート
- [ ] 複数ジャンル横断比較

## 7. 除外事項

| 項目 | 理由 |
|------|------|
| amazon.com（英語市場）対応 | スコープ外・別プロジェクト |
| KENP（KU既読ページ数）直接取得 | KDPダッシュボードのみアクセス可能 |
| Playwright/Seleniumスクレイピング | Amazon ToS違反・2025年11月提訴実績あり |
| 書影画像の直接ホスティング | Amazonの著作権ポリシー違反 |
| リアルタイム（1秒以内）BSR更新 | Keepa APIの制限（20トークン/分）上 |
