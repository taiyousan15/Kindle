### トレンド検知アルゴリズム

```
TrendScore = (
  0.40 * normalized(stars_7d_delta / total_stars) +    // 急増率
  0.30 * normalized(npm_28d_growth_rate) +              // DL成長率
  0.20 * normalized(hn_score + reddit_score) +           // コミュニティ言及
  0.10 * recency_boost                                   // 新着ブースト
)
```

### セマンティック検索設計

## キーワードスキルが最初に必要な理由

```
旧フロー（問題あり）:
  PHASE 0: intelligence-research
  PHASE 1: リサーチ開始
           ↓
    検索クエリが固定・手書き → 抜け漏れが出る

正しいフロー（v4.0）:
  PHASE 0 ステップ1: keyword-mega-extractor
           ↓
    テーマから関連語・複合語・ニッチ語・急上昇語を展開
           ↓
  PHASE 0 ステップ2: intelligence-research（並行）
           ↓
  PHASE 1: 展開されたキーワードでリサーチ
           → カバレッジが格段に上がる
```

## existing agents.json との対応

既存の `keyword-miner` エージェントと `keyword-mega-extractor` スキルの役割：

| 要素 | 場所 | 役割 |
|------|------|------|
| `keyword-miner` agent | agents.json（既存） | サブエージェントとして詳細キーワード抽出 |
| `/keyword-mega-extractor` | skill_map（新規追加） | Phase 0の最初に即時展開 |
| `keyword_mining` | methodology（既存） | 手順・タクソノミ定義 |

→ 3つが連携することで、シード段階から網羅的なキーワード宇宙を構築できる。
